# Key Architectural Decisions

## Decision 1: Flat Structure vs. Nested Structure
**Date:** 2026-07-07
**Decision:** Use flat structure (all files in `src/`)
**Rationale:** Simplified imports, no circular dependencies
**Status:** Implemented

## Decision 2: MDIPS as the Semantic Standard
**Date:** 2026-07-07
**Decision:** DBIP will implement MDIPS v3.0
**Rationale:** Ensures ecosystem interoperability
**Status:** Pending implementation

## Decision 3: Multi-Dimensional Confidence
**Date:** 2026-07-07
**Decision:** Use multi-dimensional confidence from MDIPS/FICS
**Rationale:** Single scores are insufficient for intelligence
**Status:** Pending implementation

## Decision 4: ISCA as Communication Layer
**Date:** 2026-07-07
**Decision:** DBIP communicates through ISCA
**Rationale:** Protocol independence, loose coupling
**Status:** Planned
